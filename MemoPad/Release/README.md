# Release

Build に成功すると, ココに置いてある .exe のようなものが出来上がります.
オブジェクトサイズが 180 KB と出ていますね. ( 2023/11/26 現在 )
小っさいです. 元祖 Notepad.exe のサイズに比べると, ずいぶん小っさいと言えましょう.

ここまで小っさいのは, 
.dll として切り離せるものは .exe から切り離しているからで,
この .exe だけをダウンロードして動かそうとすると,
「[DLL](https://ja.wikipedia.org/wiki/ダイナミックリンクライブラリ) がありません」とか言われることがあります.
そういう場合は, その DLL の版元である Microsoft&reg; さんの
「[DLL をタダでくれるというサイト](https://learn.microsoft.com/ja-jp/cpp/windows/latest-supported-vc-redist?view=msvc-170)」から,
「Visual Studio 2022」の「X64」の DLL をセットアップしてくれる .exe をダウンロードして実行する必要があります.

なんでそんな「外付けの .dll が要る .exe」なんて意地悪な構成にするのかというと, 下記の利点があるからです.

* オブジェクトサイズが小さくなるので, SSD ( HDD ) の容量節約になる.
* 同じ DLL を使う複数のプロセスが居る場合は DLL が共有されるので, メモリーの節約にもなる.
* Microsoft&reg; さんの DLL には Microsoft&reg; さんがセキュリティー更新をかけてくれるので, 今後も安心.

ただし, DLL 云々の話以前に, .exe のダウンロードを試みた時点で,
良心的なブラウザと OS は
「やめとこ. ね?」とか「削除するよ? いいでしょ?」とか「ホントに起動するの? 正気?」とか,
あなたの PC を守ろうとします.
これに打ち勝つには相当の意志の強さと提供者への信頼と PC の動作に対する観察力が要りますが,
ダウンロードできちゃうことはできちゃうのです.
( 少なくとも Microsoft Edge の場合はできちゃいました. )

この Relase フォルダの目的は,
冒頭で述べたようにオブジェクトサイズの規模を例示することにあるのです.
しかし, 強い意志を貫いてダウンロードを完遂した人々の, その後の .exe の利用をさまたげようとも思いません.

**でも,
ここから .exe を単品でダウンロードするより,
[Releases](https://github.com/inhouse-tool/MemoPad/releases) から .msi をダウンロードした方がラクです.**
少なくとも .exe 単品のダウンロードを試みた場合と違って OS やブラウザがそれほどイヤがらないので, 気分的にもラクです.
MemoPad.msi をクリックした先の画面で,
左端にある「受け皿に矢印が落ちているような絵柄」のアイコンをつついてダウンロードしてください.
ダウンロードした .msi をダブルクリックすると, ごく普通の手順でインストールできます.
( インストールがうまくいかない場合は,
[管理者権限が足りない](https://www.google.com/search?q=msi+管理者権限)ことが多いそうです. )

なお, この .msi にも DLL は含まれていません.
インストール後起動して「DLL がありません」とか言われた場合は,
上にあるように「X64」の DLL をセットアップしてください.
お手数をおかけしますが, どうやらこれが最善の提供形態のようなんです.
